# End Product Definition

## Product category

ProPokerTV is the non-gambling poker media and reputation layer for creators, players, clubs, events, brands, and fans.

## MVP wedge

The first wedge is:

> The weekly poker clip league.

Creators post poker clips. Fans vote. Winners earn status. Leaderboards and badges turn clips into reputation.

## End product

ProPokerTV becomes a non-gambling poker media ecosystem where:

- creators build poker identity and audience,
- fans discover, vote, comment, react, and follow poker moments,
- clubs and events run highlight contests,
- leaderboards and badges create status,
- brands sponsor contests and creator opportunities,
- poker moments become reputation, rankings, community, and commercial opportunity.

## Core loops

### Creator loop

Creator posts clip → gets votes/views → earns ranking/status → posts again.

### Fan loop

Fan watches clip → votes/comments/reacts → influences leaderboard → returns next week.

### Competition loop

Weekly contest → nominations → voting → winner → leaderboard history → repeat.

### Reputation loop

Creator history → badges/rankings → credibility → sponsor/club opportunities.

### Partner loop

Club/event/brand sponsors contest → gets visibility/analytics/leads → repeats.

## Actor map

| Actor | Wants | ProPokerTV must provide |
|---|---|---|
| Fans | Entertainment, drama, learning, influence | Feed, contests, voting, rankings |
| Creators | Visibility, status, audience, income | Creator profile, clip posting, leaderboards, badges |
| Players | Recognition, documented poker identity | Player profile, clips, achievements |
| Clubs/events | Community activation, marketing, highlight distribution | Branded contest pages, event highlight feed, dashboards |
| Brands | Credible niche audience | Sponsored contests, creator partnerships |
| Admin/moderators | Safety and quality | Moderation queue, report review, permissions |

## End-state product definition of done

The end product is not done until ProPokerTV can support:

1. A creator joining, building a profile, posting clips, and earning persistent reputation.
2. A fan discovering active contests, voting safely, reacting/commenting, and returning for the next week.
3. An admin/moderator approving clips, managing nominations, finalizing contests, and protecting quality.
4. Completed contests generating durable winner history, creator stats, and leaderboard updates.
5. At least one partner/club/event pilot using ProPokerTV as a contest or highlight surface.
6. The platform remaining clearly non-gambling and brand-safe.

## Not end product

ProPokerTV is not complete if:

- it is only a generic clip feed,
- weekly contest is a side feature,
- winners do not create persistent reputation,
- creator profile does not show status,
- voting is easy to abuse,
- moderation is not operational,
- a new developer cannot run the app from docs,
- there is no measurable creator/fan loop.
